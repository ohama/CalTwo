**Test 1: Dev Server Startup**

1. Open terminal in project root
2. Run: `npm run dev`
3. Expected: Server starts without errors, shows URL (http://localhost:5173/)
4. Open browser to http://localhost:5173/
5. Expected: Page shows "Hello CalTwo" heading and "F# + Elmish + Vite + HMR is working!" text

**Test 2: HMR (Hot Module Replacement)**

1. Keep dev server running from Test 1
2. Open `src/App.fs` in editor
3. Find line: `{ Message = "Hello CalTwo" }`
4. Change to: `{ Message = "HMR 동작 확인!" }`
5. Save file
6. Expected: Browser updates automatically WITHOUT refresh
7. Expected: New message "HMR 동작 확인!" appears immediately
8. Expected: No page flicker or state loss

**Test 3: Source Maps**

1. Keep dev server running
2. Open browser DevTools (press F12)
3. Go to Sources tab
4. Look for file tree on left side
5. Navigate to: localhost:5173 → src → App.fs
6. Expected: Original F# code is visible (NOT transpiled JavaScript)
7. Expected: Syntax highlighting shows F# keywords (let, type, match)

**Test 4: Version Pinning**

1. Run: `cat global.json`
2. Expected: Shows "version": "10.0.100"
3. Run: `cat package.json | grep react`
4. Expected: Shows "react": "18.3.1" (NOT React 19)
5. Run: `cat src/App.fsproj | grep Fable`
6. Expected: Shows exact versions (4.3.0, 5.0.2, etc.)

**Test 5: Korean Tutorial**

1. Open: `tutorial/phase-01.md`

3. Expected: Contains sections explaining MVU pattern, Fable, Elmish, HMR
4. Expected: Includes code examples for all files (App.fs, Main.fs, vite.config.js)
5. Expected: Has troubleshooting section (자주 발생하는 문제)
